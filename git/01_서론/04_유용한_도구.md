# 유용한 Git 도구

Git은 명령줄 도구 자체로도 강력하지만, 함께 사용하면 더욱 생산성을 높여주는 다양한 도구들이 있습니다. 이 장에서는 Git 사용을 더 편리하고 효율적으로 만들어주는 유용한 도구들을 소개합니다.

## 학습 목표

- Git 관련 유용한 도구의 종류와 특징을 이해한다
- `tig`를 활용하여 터미널에서 효율적으로 Git 히스토리를 탐색할 수 있다
- 상황에 맞는 적절한 Git 도구를 선택하여 사용할 수 있다

## 1. tig: 터미널용 Git 브라우저

`tig`는 터미널에서 동작하는 텍스트 모드 Git 브라우저입니다. 커밋 로그, 브랜치, diff 등을 키보드만으로 빠르게 탐색할 수 있습니다. 마치 `git log --oneline --graph --all`을 대화형으로 사용하는 것과 같습니다.

### 설치

```bash
# macOS
$ brew install tig

# Ubuntu/Debian
$ sudo apt install tig

# Windows (Git Bash)
$ choco install tig
```

### 실행 화면 예시

```text
$ tig

  Browse branch 'main'                           +14:30
  [main] a1b2c3d Add login page                  홍길동 2 min ago
  [main] d4e5f6f Fix payment bug                 홍길동 1 hour ago
  [main] g7h8i9j Add payment module              홍길동 3 hours ago
  [main] k1l2m3n Initial commit                  홍길동 1 day ago
  [main] a1b2c3d Add login page
  ─────────────────────────────────────────────────────────
  diff --git a/login.html b/login.html
  new file mode 100644
  @@ -0,0 +1,10 @@
  +<h1>Login</h1>
  +<form>
  +  <input type="text" name="username">
  +  <input type="password" name="password">
  +  <button>Submit</button>
  +</form>

  [main] a1b2c3d - Add login page               +14:30
  [status] [logs] [diff] [tree] [blame] [help]
```

위 화면에서 `↓`/`↑` 키로 커밋을 선택하고 `Enter`를 누르면 diff를 볼 수 있습니다. `l` 키를 누르면 전체 로그 뷰로, `t` 키를 누르면 트리 뷰(파일 목록)로 전환됩니다.

### 주요 사용법

```bash
# 커밋 로그 브라우저 열기 (가장 많이 사용)
$ tig

# 특정 파일의 히스토리 보기
$ tig -- README.md

# 브랜치 간 비교
$ tig main..feature/login

# Stash 목록 보기
$ tig stash

# grep 결과를 tig로 보기
$ tig grep "function"
```

### tig 단축키

`tig`를 실행한 후 사용할 수 있는 주요 키보드 단축키입니다:

| 키 | 기능 |
|---|---|
| `j` / `k` | 위/아래로 이동 |
| `Enter` | 선택한 커밋의 상세 내용(diff) 보기 |
| `l` | 로그 뷰로 전환 |
| `d` | diff 뷰로 전환 |
| `t` | 트리 뷰로 전환 (파일 목록) |
| `/` | 검색 |
| `m` | 현재 브랜치에서 merge |
| `M` | rebase |
| `r` | refresh (새로고침) |
| `q` | 뒤로 가기 / 종료 |
| `h` | 도움말 |

### 실전 예시

```bash
$ cd my-project
$ tig
# ↓ j 키로 로그 탐색
# ↓ Enter 키로 diff 확인
# ↓ / 키로 "버그" 검색
# ↓ n 키로 다음 검색 결과로 이동
```

## 2. lazygit: 터미널용 Git GUI

`lazygit`는 터미널에서 동작하는 더 직관적인 Git GUI 도구입니다. 화살표 키로 Staging, 커밋, 브랜치 전환, 병합 등을 모두 처리할 수 있습니다.

```bash
# 설치
# macOS
$ brew install lazygit

# Ubuntu/Debian
$ sudo add-apt-repository ppa:lazygit-team/release
$ sudo apt update
$ sudo apt install lazygit

# 실행
$ lazygit
```

### 실행 화면 예시

```text
  ┌─ Status ─────────────────────────────┐ ┌─ Staging Area ────────────────────┐
  │  ↑↓ my-project                       │ │                                   │
  │                                      │ │   ■ M index.html                  │
  │  ? ─ Recent repositories             │ │   ■ M style.css                   │
  │                                      │ │                                   │
  │  ┌─ Commits ─────────────────────┐   │ │                                   │
  │  │ a1b2c3d Add login page        │   │ │                                   │
  │  │ d4e5f6f Fix payment bug       │   │ │                                   │
  │  │ g7h8i9j Add payment module    │   │ └───────────────────────────────────┘
  │  │ k1l2m3n Initial commit        │   │ ┌─ Main ────────────────────────────┐
  │  └───────────────────────────────┘   │ │  1: unstaged changes              │
  │                                      │ │  ?? index.html                    │
  │  ┌─ Branches ───────────────────┐    │ │  ?? style.css                     │
  │  │ * main                       │    │ └───────────────────────────────────┘
  │  │   feature/login              │    └──────────────────────────────────────
  │  └──────────────────────────────┘
```

화살표 키로 파일을 선택하고 `Space`로 Staging/Unstaging, `c`로 커밋 메시지 작성, `p`로 push까지 한 화면에서 처리할 수 있습니다.

## 3. gh: GitHub CLI

`gh`는 GitHub을 명령줄에서 제어할 수 있는 공식 CLI 도구입니다.

```bash
# 설치
# macOS
$ brew install gh

# Ubuntu/Debian (GitHub CLI 공식 저장소)
$ sudo apt install gh

# 로그인
$ gh auth login
```

### 실행 화면 예시

```text
$ gh pr list --limit 3
Showing 3 of 3 open pull requests in jingu279/Document

#51  Add login form              feature/login     about 2 hours ago
#49  Fix payment calculation bug  hotfix/payment     about 1 day ago
#48  Update README                docs/readme        about 3 days ago

$ gh pr view 51
Add login form ─ #51
Open • feature/login → main • about 2 hours ago
Assignee: jingu279 • 1 reviewer requested

Adds login form with email and password fields.
Includes validation and error handling.

+120 -0 in 3 files
  A login.html      +45 -0
  A login.css       +35 -0
  A login.js        +40 -0

$ gh issue list
Showing 5 of 5 open issues in jingu279/Document

#12  Header styling broken            bug         about 1 day ago
#11  Add search functionality         enhancement  about 2 days ago
#10  Update dependencies              chore        about 3 days ago
```

### 주요 명령어

```bash
$ gh repo create my-project    # GitHub에 저장소 생성
$ gh pr create -f              # PR 생성 (자동 제목)
$ gh pr review --approve       # PR 승인
$ gh pr merge                  # PR 병합
$ gh run watch                 # Actions 실행 실시간 확인
$ gh issue list --label bug    # 특정 라벨의 이슈 검색
```

## 4. diff-so-fancy: 예쁜 diff

`diff-so-fancy`는 `git diff`의 출력을 더 읽기 쉽게 보여주는 도구입니다.

```bash
# 설치
$ brew install diff-so-fancy   # macOS
$ npm install -g diff-so-fancy  # npm

# Git 설정에 적용
$ git config --global core.pager "diff-so-fancy | less --tabs=4 -RFX"
$ git config --global interactive.diffFilter "diff-so-fancy --patch"
```

### 적용 전후 비교

```text
# 적용 전 (기본 git diff):
─────────────────────────────────────────────────────────
diff --git a/index.html b/index.html
index a1b2c3d..d4e5f6g 100644
--- a/index.html
+++ b/index.html
@@ -10,7 +10,9 @@
     <title>내 웹사이트</title>
   </head>
   <body>
+    <h1>환영합니다!</h1>
     <p>내용입니다.</p>

# 적용 후 (diff-so-fancy):
─────────────────────────────────────────────────────────
index.html ──────────────────────────────────────────────
@¡@ 10,7 10,9 @¡@
     <title>내 웹사이트</title>
   </head>
   <body>
+    <h1>환영합니다!</h1>
     <p>내용입니다.</p>
```

변경된 파일명이 강조 표시되고, 청크 헤더가 간결해져서 가독성이 크게 향상됩니다.

## 5. gitk: GUI 히스토리 뷰어

`gitk`는 Git에 내장된 그래픽 히스토리 뷰어입니다. 별도 설치가 필요 없습니다.

```bash
$ gitk
```

### 실행 화면 예시

```text
  ┌─────────────────────────────────────────────────────────────┐
  │  [●] a1b2c3d  Add login page                 main,* HEAD   │
  │  │                                                     │   │
  │  [●] d4e5f6f  Fix payment bug                             │   │
  │  │                                                     │   │
  │  [●] g7h8i9j  Add payment module            feature/pay   │   │
  │  │/                                                      │   │
  │  [●] k1l2m3n  Initial commit                             │   │
  │                                                          │   │
  │  ────────────────────────────────────────────────────    │   │
  │  a1b2c3d Add login page                                  │   │
  │                                                          │   │
  │  diff --git a/login.html b/login.html                    │   │
  │  index 0000000..e69de29                                  │   │
  │  --- /dev/null                                           │   │
  │  +++ b/login.html                                        │   │
  └─────────────────────────────────────────────────────────────┘
```

상단 패널에서 히스토리 그래프를 시각적으로 확인하고, 하단 패널에서 선택한 커밋의 diff를 볼 수 있습니다. 마우스 클릭으로 탐색할 수 있어 초보자에게 직관적입니다.

## 6. git-gui: GUI 커밋 도구

`git-gui`는 Git에 내장된 GUI 도구로, Staging과 커밋을 시각적으로 수행할 수 있습니다.

```bash
$ git gui
```

```text
  ┌─ Unstaged Changes ────────┐ ┌─ Staged Changes ──────────┐
  │  index.html               │ │  style.css                │
  │  app.js                   │ │                           │
  │                           │ │                           │
  │  [>>] Stage                │ │  [<<] Unstage             │
  └───────────────────────────┘ └───────────────────────────┘
  ┌─ Commit Message ────────────────────────────────────────┐
  │  [로그인 페이지 추가]                                      │
  │                                                          │
  │  [Commit]  [ Amend ]  [ Sign Off ]                       │
  └──────────────────────────────────────────────────────────┘
  ┌─ Diff (index.html) ─────────────────────────────────────┐
  │  +<h1>환영합니다!</h1>                                   │
  └──────────────────────────────────────────────────────────┘
```

왼쪽에서 unstaged 파일을 선택하고 Stage 버튼을 누른 후, 커밋 메시지를 입력하고 Commit 버튼을 누르면 됩니다.

## 7. git-secrets: 보안 검사

`git-secrets`는 커밋에 비밀 키나 API 토큰이 포함되지 않도록 방지하는 도구입니다. AWS, GCP 등의 패턴이 기본 내장되어 있습니다.

```bash
# 설치
$ brew install git-secrets

# 저장소에 적용
$ git secrets --install

# AWS 키 패턴 등록
$ git secrets --register-aws
```

```text
$ git commit -m "config 추가"
   ┌─────────────────────────────────────────┐
   │  [ERROR]                                 │
   │  login.html:5:  AKIAIOSFODNN7EXAMPLE      │
   │                                          │
   │  ⚠ AWS Access Key ID found in commit!   │
   │                                          │
   │  파일에서 하드코딩된 키를 제거하거나     │
   │  git secrets --add 로 허용 패턴을 등록    │
   │  하세요.                                  │
   └─────────────────────────────────────────┘
   커밋이 차단되었습니다.
```

## 한눈에 정리

| 도구 | 용도 | 설치 방법 |
|---|---|---|
| **tig** | 터미널 Git 브라우저 (히스토리 탐색) | `brew install tig` |
| **lazygit** | 터미널 Git GUI (직관적 조작) | `brew install lazygit` |
| **gh** | GitHub CLI | `brew install gh` |
| **diff-so-fancy** | diff 출력 가독성 향상 | `brew install diff-so-fancy` |
| **gitk** | GUI 히스토리 뷰어 (내장) | Git 설치 시 포함 |
| **git-gui** | GUI Staging/커밋 (내장) | Git 설치 시 포함 |
| **git-secrets** | 비밀 키 노출 방지 | `brew install git-secrets` |
