# 유용한 Git 도구

Git은 명령줄 도구 자체로도 강력하지만, 함께 사용하면 더욱 생산성을 높여주는 다양한 도구들이 있습니다. 이 장에서는 Git 사용을 더 편리하고 효율적으로 만들어주는 유용한 도구들을 소개합니다.

## 학습 목표

- Git 관련 유용한 도구의 종류와 특징을 이해한다
- `tig`를 활용하여 터미널에서 효율적으로 Git 히스토리를 탐색할 수 있다
- 상황에 맞는 적절한 Git 도구를 선택하여 사용할 수 있다

## 1. tig: 터미널용 Git 브라우저

`tig`는 터미널에서 동작하는 텍스트 모드 Git 브라우저입니다. 커밋 로그, 브랜치, diff 등을 키보드만으로 빠르게 탐색할 수 있습니다.

### 설치

```bash
# macOS
$ brew install tig

# Ubuntu/Debian
$ sudo apt install tig

# Windows (Git Bash)
$ choco install tig
```

### 주요 사용법

```bash
# 커밋 로그 브라우저 열기 (가장 많이 사용)
$ tig

# 특정 파일의 히스토리 보기
$ tig -- README.md

# 브랜치 간 비교
$ tig main..feature/login

# Stash 목록 보기
$ tig stash

# grep 결과를 tig로 보기
$ tig grep "function"
```

### tig 단축키

`tig`를 실행한 후 사용할 수 있는 주요 키보드 단축키입니다:

| 키 | 기능 |
|---|---|
| `j` / `k` | 위/아래로 이동 |
| `Enter` | 선택한 커밋의 상세 내용(diff) 보기 |
| `l` | 로그 뷰로 전환 |
| `d` | diff 뷰로 전환 |
| `t` | 트리 뷰로 전환 (파일 목록) |
| `/` | 검색 |
| `m` | 현재 브랜치에서 merge |
| `M` | rebase |
| `r` | refresh (새로고침) |
| `q` | 뒤로 가기 / 종료 |
| `h` | 도움말 |

### 실전 예시

```bash
$ cd my-project
$ tig
# ↓ j 키로 로그 탐색
# ↓ Enter 키로 diff 확인
# ↓ / 키로 "버그" 검색
# ↓ n 키로 다음 검색 결과로 이동
```

`tig`는 `git log --oneline --graph --all`을 키보드로 실시간 탐색할 수 있게 해줍니다. 터미널을 벗어나지 않고 빠르게 히스토리를 확인할 수 있어 많은 개발자가 선호합니다.

## 2. lazygit: 터미널용 Git GUI

`lazygit`는 터미널에서 동작하는 더 직관적인 Git GUI 도구입니다. 화살표 키로 Staging, 커밋, 브랜치 전환, 병합 등을 모두 처리할 수 있습니다.

```bash
# 설치
# macOS
$ brew install lazygit

# Ubuntu/Debian
$ sudo add-apt-repository ppa:lazygit-team/release
$ sudo apt update
$ sudo apt install lazygit

# 실행
$ lazygit
```

lazygit의 장점은 Git 명령어를 외우지 않아도 화살표 키만으로 대부분의 작업을 수행할 수 있다는 점입니다. 초보자에게 특히 유용합니다.

## 3. gh: GitHub CLI

`gh`는 GitHub을 명령줄에서 제어할 수 있는 공식 CLI 도구입니다.

```bash
# 설치
# macOS
$ brew install gh

# Ubuntu/Debian (GitHub CLI 공식 저장소)
$ sudo apt install gh

# 로그인
$ gh auth login

# 주요 명령어
$ gh repo create my-project    # GitHub에 저장소 생성
$ gh pr create                  # PR 생성
$ gh pr review                  # PR 리뷰
$ gh issue list                 # 이슈 목록
$ gh run watch                  # Actions 실행 확인
```

`gh`는 GitHub를 자주 사용한다면 반드시 설치해야 할 도구입니다. GitHub 웹 UI를 열지 않고도 터미널에서 모든 작업을 처리할 수 있습니다.

## 4. diff-so-fancy: 예쁜 diff

`diff-so-fancy`는 `git diff`의 출력을 더 읽기 쉽게美化해주는 도구입니다.

```bash
# 설치
$ brew install diff-so-fancy   # macOS
$ npm install -g diff-so-fancy  # npm

# Git 설정에 적용
$ git config --global core.pager "diff-so-fancy | less --tabs=4 -RFX"
$ git config --global interactive.diffFilter "diff-so-fancy --patch"
```

## 5. git-secrets: 보안 검사

`git-secrets`는 커밋에 비밀 키나 API 토큰이 포함되지 않도록 방지하는 도구입니다.

```bash
# 설치
$ brew install git-secrets
$ git secrets --install
$ git secrets --register-aws  # AWS 키 패턴 등록
```

## 한눈에 정리

| 도구 | 용도 | 설치 명령어 |
|---|---|---|
| **tig** | 터미널 Git 브라우저 (히스토리 탐색) | `brew install tig` |
| **lazygit** | 터미널 Git GUI (직관적 조작) | `brew install lazygit` |
| **gh** | GitHub CLI (터미널에서 GitHub 제어) | `brew install gh` |
| **diff-so-fancy** | diff 출력 가독성 향상 | `brew install diff-so-fancy` |
| **git-secrets** | 비밀 키 노출 방지 | `brew install git-secrets` |
| **gitk** | GUI 히스토리 뷰어 (Git 내장) | Git 설치 시 포함 |
| **git-gui** | GUI Staging/커밋 도구 (Git 내장) | Git 설치 시 포함 |
